# 排班表 Excel → Txt 翻译

## 适用场景

- 用户提供了一份基建排班 Excel 表格，需要翻译为项目中的 `src/assets/texts/schedule/` 下的排班 txt 文件
- 需要根据 Excel 中的干员、效率、队列等信息生成符合解析器格式的 txt

## Excel 排班表格式

### 列结构

每列依次为：`[标签]` `[队列1_干员]` `[队列1_%]` `[队列2_干员]` `[队列2_%]` `[队列3_干员]` `[队列3_%]`

即每两个列为一组队列：第一个是干员名，第二个是效率百分比。

### 多行站点

当一个站点（如制造站、贸易站）有多个干员时，第一行写第一个干员及其效率，后续行补充更多干员（后续行的效率列为空）。

```
制造站 1
作战记录	远牙	42%	35	35%	远牙	42%
	野鬃	42%	35	35%	野鬃	42%
	灰毫	42%	35	35%	灰毫	42%
```

### 配对列结构（控制中枢、宿舍）

对于需要配对摆放的站点（控制中枢、宿舍），每队列占两列，格式为：

```
控制中枢	阿米娅	14% 贸易	斩业星熊	15% 制造	八幡海铃
	八幡海铃		诗怀雅	14% 贸易	诗怀雅	14% 贸易
```

### 百分比的含义

- 每个干员右侧的百分比是其单人效率贡献
- 如果某干员右侧没写百分比（如 `多萝西` 单独出现），则该效率属于整组
- 整组效率示例：`巫恋 90%` 代表巫恋+裁缝+龙舌兰整组效率 90%

## Txt 排班表格式

### 文件结构

```
标题行
子标题行
-=-
描述行（可多行）
-=-
统计数据行
队列说明行
站点1标题
站点1类型
队列1干员 | 描述
队列2干员 | 描述
队列3干员 | 描述
---
站点2标题
站点2类型
...
===
另一区块
...
```

### 分隔符

- `-=`：分隔标题/描述/数据
- `---`：分隔同一区块内的不同站点
- `===`：分隔不同区块

### 干员格式

```
干员名精英等级    例如：能天使2、红云1、煌0
Lancet-2 需要写为 Lancet-20!（带叹号表示心情耗尽）
```

### 队列格式

每行一个队列，干员用空格分隔，`|` 后接描述：

```
远牙2 野鬃2 灰毫2 | 126%
阿米娅0 八幡海铃2 焰尾2 |
```

## 翻译规则

一些tips：

- excel里的裁缝在txt里改成”柏喙2“
- excel里的作战记录制造站中的”35”表示任意35%作战记录的干员，在txt里需要替换为实际的干员。有这些干员：断罪者1、食铁兽2、酒神2、弑君者2、裂响2。你需要确保每个干员最多工作2个队列，并且同一个队列不能出现重复的干员。
- excel里的源石碎片制造站中的"35”表示任意35%源石碎片效率的干员，有这些干员：艾雅法拉0、炎熔1、地灵1、锡兰2。你需要确保每个干员最多工作2个队列（除非是被菲亚梅塔007的干员）。
- 发电站空白的20%是任意干员，需替换为实际干员，可选的干员有：“格雷伊0、澄闪2、雷蛇2、炎狱炎熔2”
- “莱茵生命”为任意莱茵生命干员，需替换为”白面鸮0、梅尔0、赫默0“
- 加工站/训练室、宿舍里面放的“煌、逻各斯、迷迭香、电弧、烛煌”都是精0即可，即写”煌0“等，除非这里面的干员出现在了其他站中（比如，如果迷迭香出现在了制造站中，那么迷迭香要写2）
- 除了上面我说的之外，阿米娅、夕、诗怀雅、Lancet-2、斥罪都是精0就行，砾精1就行。
- Lancet-2需要写"Lancet-20!"
- excel中，菲亚梅塔以及其右侧那个干员不需要写进txt中。

### 排除规则与 description 更新

**菲亚梅塔及其右侧配对干员** 不写入宿舍站点中。
示例：`菲亚梅塔	但书` → 两者都不写。

**同时需要更新 description 行**：description 中 `菲亚梅塔 007` 后面的干员列表，正是宿舍 Excel 中菲亚梅塔**右侧的所有配对干员**（去重后按顺序列出，中文顿号分隔）。

例如宿舍 Excel：

```
宿舍	菲亚梅塔	八幡海铃	菲亚梅塔	但书
```

菲亚梅塔的右侧配对干员为：八幡海铃（Q1）和 但书（Q2），所以 description 写：

```
菲亚梅塔 007 八幡海铃、但书
```

注意菲亚梅塔和这些干员**只在宿舍站点中排除**，description 中仍然保留。

### 占位符干员分配规则

以作战记录制造站中的 `35` 占位符为例。

从池子 `断罪者1、食铁兽2、酒神2、弑君者2、裂响2` 中分配：

- 每个干员最多工作 **2 个队列**
- **同一个队列内不能出现重复干员**
- 优先填满制造站1的Q2（3人），再填充制造站2（每队2人）

### 效率标注规则

- 每个队列的 `|` 后标注效率
- 单个干员有明确效率的 → 累加后写本站总效率，如 `| 126%`
- 如果 Excel 里的效率有小数点，则四舍五入到整数百分比，并写 `| 平均 x%`
- 当龙舌兰 / 可露希尔 / 但书出现在贸易站时，需要在描述中注明“龙舌兰组/可露希尔组/但书组，纸面 x%”。如果龙舌兰和但书同时出现在贸易站，则需要在描述中注明“龙舌兰但书组，纸面 x%”。
- 有特殊情况的 → 写说明文字，如 `| 爱丽丝、车尔尼放高等级宿舍`

### 描述文字保留

原 txt 中的部分描述文字（如 `令心情 < 12，夕心情 ≥ 12`、`爱丽丝、车尔尼放高等级宿舍`）需根据 Excel 中对应的条件保留或更新。

### 产出行（Stats 行）计算

排班 txt 的第一段 `-=` 之后有一行统计数据，格式为：

```
EXP 31.7k + 5.5k | 贵金属 61.2k + 6.9k | 龙门币 53.1k + 12.7k | 高级凭证 0.843
```

其中各字段从 Excel 底部的 **产出表** 中提取：

| 字段                 | Excel 参考行   | 参考列          | 处理方式                                    |
| -------------------- | -------------- | --------------- | ------------------------------------------- |

### 9. 排班表一致性校验

翻译完成后，需要检查 Excel 排班表是否满足以下约束条件。如果违反，需反馈给用户确认。

#### 规则 1：同队列无重复干员

**每个队列内部不能出现同一名干员两次**（无论精英等级是否相同）。

例如：队列 1 中不能同时出现两个"断罪者"。

#### 规则 2：被菲亚梅塔 007 的干员需出现在每个队列中

description 中 `菲亚梅塔 007` 后面的干员（即菲亚梅塔在宿舍中的配对干员），**必须在每个队列中都出现一次**（可以在不同站点）。

例如 description 为 `菲亚梅塔 007 但书、苍苔`，则但书和苍苔在每个队列中都必须出现恰好一次。

#### 规则 3：非 007 干员不能全勤

**除被菲亚梅塔 007 的干员之外**，其他所有干员**不能在每个队列中都出现**。至少有一个队列中该干员不在上班（即休息）。宿舍中的干员算休息，不算上班。

例如：如果八幡海铃在 Q1、Q2 中都出现了，但在 Q3 中只出现在宿舍，不算违反规则；如果八幡海铃在 Q1、Q2、Q3 的生产站中都有出现，则违反规则。

### 10. 未知干员与可疑数据检查

翻译过程中，如果遇到任何矛盾、可疑之处（包括但不限于以下情况），**必须向用户指出并请求确认**，不能直接相信 Excel。遇到任何不确定的情况都要向用户确认，而不是直接翻译或修改 Excel 数据。以下是一些需要特别注意的情况：

#### 需要向用户指出的情况

1. **Excel 中出现了未知干员或者未知的占位符** → 询问用户是否需要添加到 SKILL.md 中，并询问该干员的精英等级和效率
2. **Excel 中干员右侧没有标注效率，但 SKILL.md 中有固定值** → 指出并让用户确认
   - 例如：可露希尔在 Excel 中右侧空白，但 SKILL.md 中可露希尔为固定 10%，需指出并询问用户是否遗漏
3. **Excel 中的干员效率与"干员效率与精英等级参考表"中的值不一致** → 指出差异，让用户确认
   - 例如：Excel 中某干员效率为 40%，但“干员效率与精英等级参考表”中该干员效率为 35%，需指出差异并询问用户
4. **干员出现在不合理的站点** → 指出差异，让用户确认
   - 例如：阿罗玛是赤金制造站干员，如果 Excel 中阿罗玛出现在作战记录制造站，则需要指出并询问用户
   - 例如：但书是贸易站干员，如果 Excel 中但书出现在其他站点，则需要指出并询问用户
5. **效率计算公式可能与其他干员联动** → 检查联动条件是否满足
   - 例如：砾的效率和苍苔同站、薇薇安娜在控制中枢有关；引星棘刺和贸易站数量、苍苔同站有关，需仔细检查干员效率是否正确
